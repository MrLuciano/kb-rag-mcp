     def on_deleted(self, event):
         """Handle file deletion — remove vectors from Qdrant."""
         if self.should_ignore(event.src_path):
             return
         log.info(f"File deleted: {event.src_path}")
         if self.delete_handler is not None:
             try:
                 self.delete_handler(event.src_path)

     # VectorStore reads QDRANT_COLLECTION, QDRANT_HOST, QDRANT_PORT from env

     vector_store = VectorStore()

     # IngestRegistry tracks per-file status (has mark_deleted); MetadataStore
     # (already in scope as `store`) does not have that method.
     registry = IngestRegistry()
     registry.connect()

     def _delete_handler(src_path: str) -> None:
         try:
             rel_path = str(Path(src_path).relative_to(watch_path))
         except ValueError:
             log.warning(

             rel_path = src_path
         try:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → DocWatcher | Watchdog delivers raw filesystem events; path may point outside watch_path |
| DocWatcher → VectorStore | Delete call removes vectors by source_file key; wrong key = silent no-op |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | on_deleted path | mitigate | `should_ignore()` already gates on extension and temp patterns; delete_handler receives raw path — caller (main) must derive relative path safely using `Path.relative_to()` inside try/except |
| T-02-02 | Denial of Service | asyncio.run in delete_handler | accept | Single blocking call per deletion event; Qdrant unavailability is caught and logged; watcher continues |
| T-02-03 | Information Disclosure | log.info with file paths | accept | Internal deployment only; no auth layer per project constraints |
</threat_model>
