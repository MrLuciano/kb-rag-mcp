# Phase 52-01: Ingestion Schedule Management

## Goal
Create CRON-based ingestion schedule management in the Admin UI Schedule tab with a background scheduler that creates Jobs on cron match.

## Tasks

### Task 1: Cron expression matcher

- **What**: Pure Python 5-field cron expression parser/matcher at `ingest/core/cron.py`
- **Read**: `ingest/core/metadata.py` (naming patterns)
- **Acceptance**: `cron_matches(datetime, expression)` returns bool for `*`, `*/N`, `N-M`, comma lists across all 5 fields; `next_cron_time(reference, expression)` returns next matching datetime

```python
def cron_matches(dt: datetime, expression: str) -> bool: ...
def next_cron_time(after: datetime, expression: str) -> Optional[datetime]: ...
```

### Task 2: Schema v6 migration + Schedule CRUD

- **What**: Add `schedules` table to `MetadataStore` (schema v6) with CRUD methods
- **Read**: `ingest/core/metadata.py` (v5 migration patterns, CRUD conventions)
- **Read**: `ingest/job/models.py` (dataclass patterns)

Table columns: `id TEXT PK, name TEXT NOT NULL, cron_expr TEXT NOT NULL, docs_path TEXT NOT NULL, product TEXT, workers INT DEFAULT 2, priority TEXT DEFAULT 'normal', clean INT DEFAULT 0, force INT DEFAULT 0, enabled INT DEFAULT 1, created_at REAL, updated_at REAL, last_run_at REAL, last_run_status TEXT, next_run_at REAL`

Schedule dataclass in `models.py`:
```python
@dataclass
class Schedule:
    id: str
    name: str
    cron_expr: str
    docs_path: str
    product: Optional[str] = None
    workers: int = 2
    priority: str = "normal"
    clean: bool = False
    force: bool = False
    enabled: bool = True
    created_at: float = 0.0
    updated_at: Optional[float] = None
    last_run_at: Optional[float] = None
    last_run_status: Optional[str] = None
    next_run_at: Optional[float] = None
```

Methods: `add_schedule()`, `list_schedules()`, `get_schedule()`, `update_schedule()`, `delete_schedule()` — all using uuid4 for id, `time.time()` for timestamps, standard MetadataStore patterns.

### Task 3: Schedules API router

- **What**: FastAPI router at `kb_server/schedules/router.py` mounted on `/api/v1/schedules`
- **Read**: `kb_server/config/router.py` (auth pattern with `_verify_config_auth`)
- **Read**: `kb_server/ui/app.py` (how routers are mounted)

Endpoints:
- `GET /api/v1/schedules` — list all schedules as JSON
- `POST /api/v1/schedules` — create schedule, validate cron expr
- `PUT /api/v1/schedules/{id}` — update schedule fields
- `DELETE /api/v1/schedules/{id}` — delete schedule

Auth via `_verify_config_auth` (API key). Validation: require name, cron_expr, docs_path; return 422 with field-level errors for invalid cron expressions.

Cron validation: call `cron_matches()` with current time — raises ValueError if expression doesn't parse.

### Task 4: Background scheduler loop

- **What**: Async task in `kb_server/server.py` that polls schedules every 30s
- **Read**: `kb_server/server.py` (metric update schedule pattern at `_schedule_metric_updates`)
- **Read**: `ingest/job/manager.py` (JobManager.create_job)

Loop logic:
1. Every 30s, query `MetadataStore.list_schedules()` filtered to `enabled=True` where `next_run_at <= now()` (or null)
2. For each match: create Job via `JobManager.create_job(docs_path, product, workers, priority, clean, force)`, update `last_run_at=now()`, `last_run_status='pending'`, compute `next_run_at` via `next_cron_time()`
3. Log schedule execution
4. On error: log, don't crash

Wire up: call `_init_schedule_checker()` in `server.py:main()` startup.

### Task 5: UI — Schedule tab

- **What**: Replace placeholder `_ingestion_schedule.html` with schedule management UI
- **Read**: `kb_server/ui/templates/admin/_ingestion_schedule.html` (current placeholder)
- **Read**: `kb_server/ui/templates/admin/_users_table.html` (modal pattern)
- **Read**: `kb_server/ui/templates/admin/tab_ingestion.html` (tab structure)
- **Read**: `kb_server/ui/templates/admin/_ingestion_monitor.html` (monitor partial)

Schedule tab layout:
- Top section: Create Schedule form (name, cron_expr, docs_path, product, workers, priority, clean, force)
- Bottom section: Table of existing schedules (name, cron, docs, enabled toggle, next_run, last_run, Edit/Delete)

Alpine component `scheduleManager` in `shell.html`:
- `data-action` event delegation (matching existing pattern)
- `loadSchedules()`, `createSchedule()`, `updateSchedule()`, `deleteSchedule()`, `toggleEnabled(schedule)`
- Edit modal for updating schedules (reuses create form pattern but with populated fields)

Monitor tab (`_ingestion_monitor.html`) — no changes needed, scheduled jobs appear automatically since they use `JobManager.create_job()`.

  4. Copy files to Docker container + restart
