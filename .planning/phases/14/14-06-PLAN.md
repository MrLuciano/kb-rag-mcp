<technical_notes>
**Why wget instead of curl:**
- qdrant/qdrant:latest is Alpine-based (busybox utilities)
- wget is included in Alpine base, curl is not
- `wget --spider -q` = silent HEAD request, perfect for healthchecks
- Exit code 0 on success, non-zero on failure (same as curl -f)

**Port architecture:**
- SSE_PORT (8765) — MCP SSE transport endpoint for LLM client connections
- Port 8080 — Health server with /health and /metrics endpoints
- Prometheus scrapes 8080:/metrics (not SSE_PORT)
- Users connect to SSE_PORT for MCP, 9090 for Prometheus UI, 3000 for Grafana

**Data directory structure:**
User's existing layout (must be preserved):
```
data/
  kb_metadata.db      ← SQLite database
  registry.db         ← SQLite database  
  qdrant/            ← Qdrant vector storage
    collection/
    meta.json
```

**Volume path substitution:**
- `${VAR:-default}` syntax is Docker Compose native (no special processing)
- Works with docker-compose v1.25.0+ (current requirement: v2.x)
- Supports relative paths (./data), absolute paths (/mnt/storage), Windows paths (C:\kb-data via WSL2)
- Default values match user's current structure

**Healthcheck timing:**
- interval: 30s = check every 30 seconds
- timeout: 10s = wait 10s for response before failing
- retries: 3 = 3 failures before marking unhealthy
- Dependency chain: qdrant (30s) → kb-rag-mcp (30s + start_period 20s) → prometheus (30s) → grafana (30s + start_period 30s)
- Total cold-start time: ~60-90 seconds until all services healthy

**Backward compatibility:**
- All defaults preserve existing behavior
- No changes required for users who don't need custom paths
- SSE_PORT defaults to 8765 (matches user's existing .env)
- DATA_DIR defaults to ./data (matches current volume mount)
- QDRANT_DATA_PATH defaults to ./data/qdrant (nested under DATA_DIR)
</technical_notes>
