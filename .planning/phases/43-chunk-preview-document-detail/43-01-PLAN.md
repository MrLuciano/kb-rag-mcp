- **HTMX partial route**: returns `document_chunks.html` fragment (no `{% extends %}`), used by progressive reveal button
- **CSP nonce pattern**: Templates use `get_nonce(request)` for inline script nonces via template global (see `app.py:50-52`, `base.html`)

         ```python
                 for r in results
             ]
             chunks.sort(key=lambda c: c["chunk_index"])
         except Exception as e:
             log.error("Failed to load chunks for document %s: %s", doc_id, e)
             chunks = []
         finally:
             await store.close()
         ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser → UI Server | User-supplied `?q=` query parameter enters Jinja2 template rendering path. Without proper escaping, this is a stored/reflected XSS vector. |
| Qdrant → UI Server | Chunk text payloads from Qdrant are user-ingested content that may contain HTML markup. Rendered via Jinja2 `| escape` filter or `highlight_term` function. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Tampering | `highlight_term` function in `app.py`, `document.html` template | mitigate | `highlight_term` uses iterative approach: `html.escape()` on text segments before wrapping in `<mark>` tags. The `<mark>` tags are hardcoded in the function, not derived from input. `re.escape()` on the query term prevents regex injection. Plain-text passages (no search query) use Jinja2 `| escape` filter. |
| T-43-02 | Denial of Service | `/ui/document/{id}/chunks` HTMX endpoint | accept | Qdrant scroll has a limit of 500 chunks per query. No pagination-based DoS amplification beyond existing Qdrant query limits. Metadata-only path (no Qdrant call) for documents with 0 chunks. |
| T-43-03 | Tampering | `document_detail` route — `source_file` filter value | accept | `source_file` is taken from the trusted SQLite metadata database (populated by ingest pipeline), not from user input. No injection vector via `doc_id` integer path param. |
| T-43-SC | Tampering | pip package installs | mitigate | No new Python dependencies introduced. All packages already in `requirements.txt`. No slopcheck override needed. |
</threat_model>
