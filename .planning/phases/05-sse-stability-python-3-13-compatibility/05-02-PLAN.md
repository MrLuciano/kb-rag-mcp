<task type="auto">
  <name>Task 1: Update CI workflow with multi-version matrix</name>
  <files>.github/workflows/ci.yml</files>
  <action>

Replace the single-version CI with a matrix strategy per D-04, D-05, D-06.

**New structure:**

```yaml
name: CI

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.11", "3.12", "3.13"]

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
          pip install pytest-cov

      - name: Run tests
        run: |
          python -m pytest tests/ --ignore=tests/e2e -q --tb=short

      - name: Coverage report
        if: always()
        run: |
          python -m pytest tests/ --ignore=tests/e2e --cov=kb_server --cov-branch --cov-report=term --tb=no -q
```

**Key decisions applied:**
- Same trigger events as current (push/PR to master) per D-05
- Python 3.11, 3.12, 3.13 in the matrix per D-04 (3.12 was confirmed in discussion)
- No nightly cron per D-06
- `python-version` used as the matrix variable (consistent with `actions/setup-python` docs)

Note: The matrix jobs are completely independent — 3.13 may fail while 3.11 and 3.12 pass. That is expected and is the desired behavior (catch regressions proactively per D-11). CI output should show each version's status independently.
