## Task 2: CLI — `kb-rag tags` Command Group

**Files:** `ingest/cli/tags.py`, `ingest/cli/main.py`

**Steps:**
1. Create `ingest/cli/tags.py` with Click command group `tags_group`
2. Implement `kb-rag tags list`:
   - Query SQLite registry for distinct tag values + counts
   - Rich table output: Tag, Count, Products, Types
   - Support `--product`, `--type`, `--vendor` filters
3. Implement `kb-rag tags update`:
   - Arguments: `--add TAGS`, `--remove TAGS`, `--replace TAGS`, `--filter FIELD=VALUE`
   - `--dry-run` shows affected document count and sample IDs
   - `--yes` bypasses confirmation
   - Calls `VectorStore.update_tags()` under the hood
4. Implement `kb-rag tags remove`:
   - Arguments: `--filter FIELD=VALUE` (e.g., `product=BadProduct`)
   - `--dry-run` preview
   - Deletes from Qdrant + registry
   - Requires `--yes` or interactive confirmation
5. Implement `kb-rag tags reingest`:
   - Arguments: `--filter FIELD=VALUE`
   - Sets `status=pending` in registry
   - Deletes Qdrant chunks for matched files
   - File watcher will pick up on next scan
6. Register `tags_group` in `ingest/cli/main.py`

**Must-haves:**
- [ ] `kb-rag tags list` shows tag counts with filters
- [ ] `kb-rag tags update --add/remove/replace` works with --dry-run
- [ ] `kb-rag tags remove` deletes from both Qdrant and registry
- [ ] `kb-rag tags reingest` sets pending + deletes chunks
- [ ] All destructive ops require confirmation (or --yes)

**Tests:**
- `test_cli_tags.py` — 8 tests: list, update dry-run, update apply, remove dry-run, remove apply, reingest, filter parsing, confirmation prompt

## Task 3: Web UI — `/admin/tags` Tab

**Files:** `kb_server/ui/routes_admin.py`, `kb_server/ui/templates/admin/`

**Steps:**
1. Add `GET /admin/tags` route in `routes_admin.py` (returns `tab_tags.html` shell)
2. Create `kb_server/ui/templates/admin/tab_tags.html`:
   - HTMX-loaded partial structure (like other tabs)
   - Sub-tabs: Browse / Bulk Edit / Re-ingest Queue
3. Create `kb_server/ui/templates/admin/_tags_table.html`:
   - Filterable table with checkboxes (reuse `_documents_table.html` pattern)
   - Columns: File, Product, Type, Version, Tags, Actions
   - Tag display as Bootstrap badges
   - Bulk toolbar: Add Tags, Remove Tags, Delete, Re-ingest
4. Create `kb_server/ui/templates/admin/_tags_bulk_editor.html`:
   - Inline form for adding/removing tags on selected rows
   - Dry-run preview button
   - Apply button with confirmation
5. Create `kb_server/ui/templates/admin/_tags_reingest_queue.html`:
   - List files with `status=pending`
   - Show last error (if any)
   - Cancel re-ingest button
6. Add HTMX partial routes:

   - `POST /admin/tags/bulk-delete` — delete selected files
   - `POST /admin/tags/bulk-reingest` — mark selected for re-ingest
7. All POST routes require `require_admin` dependency

**Must-haves:**
