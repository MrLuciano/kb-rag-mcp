    - `DenseOnlyVariant(ScoringVariant)`:
      - `__init__(self, distance_metric: str = "COSINE")` — accepts "COSINE", "DOT", "EUCLID", "MANHATTAN"
      - `search()` uses `vector_store.search()` with the provided dense vector only

    - `HybridVariant(ScoringVariant)`:
      - `__init__(self, dense_weight: float = 0.7, sparse_weight: float = 0.3, rrf_k: int = 60)`
      - `search()` uses `HybridSearcher.search()` with configurable weights and rrf_k
      - Gets the hybrid searcher via `get_hybrid_searcher()` (already existing pattern)
      - Generates sparse vector via `hybrid_searcher.generate_sparse_vector(query_text)`
      - Performs dense+sparse search with RRF fusion
      - Returns fused results
    - `RerankedVariant(ScoringVariant)`:
      - `__init__(self, base_variant: ScoringVariant, model_name: str = "cross-encoder/ms-marco-MiniLM-L-6-v2", batch_size: int = 20)`
      - `search()` first calls `base_variant.search()`, then reranks results via `CrossEncoderReranker.rerank()`
      - Uses `get_reranker()` for the global reranker instance (existing pattern)
      - Warm-up: calls `_load_model()` on the reranker before the first real query to avoid non-deterministic first-run scores (per RESEARCH.md Pitfall 3)
      - Returns reranked results with updated scores
    - `ScoringEngine` class:
      - `__init__(self, variant: ScoringVariant, vector_store: Any, dataset: GoldenDataset)`
      - `async def run_experiment(self, top_k: int = 5) -> dict[str, float]`:
        1. For each example in dataset: embed query via `get_embedding(query)`, run `variant.search(query_text, vector_store, top_k)`, collect retrieved source_file names from payload
        2. Compute IR metrics via `MetricComputer.compute_all()`
        3. Return dict with keys: `recall_at_k`, `mrr`, `ndcg_at_k`, plus `variant_name`, `distance_metric`, `dense_weight`, `sparse_weight`, `reranker_enabled`
      - `async def _warmup_reranker(self) -> None` — if variant is RerankedVariant, run a dummy query through the reranker to warm up the model
      - The engine does NOT re-ingest; it assumes the collection is already populated (scoring experiments run at query time)
    - `create_variant(name: str, **kwargs) -> ScoringVariant` factory function:
      - Accepts "dense_only", "hybrid", "reranked"
      - For "reranked", requires a `base` parameter (another variant name or ScoringVariant instance)
      - Passes kwargs to constructors
      - Raises ValueError for unknown variant names
    - `DISTANCE_METRICS` dict mapping names to `qdrant_client.models.Distance` enum values:
      - "COSINE" → `Distance.COSINE`
      - "DOT" → `Distance.DOT`
      - "EUCLID" → `Distance.EUCLID`
      - "MANHATTAN" → `Distance.MANHATTAN`

    - Import organization: stdlib → third-party → local

     - Do NOT re-implement RRF fusion; reuse `HybridSearcher._rrf_fusion()` via `HybridSearcher.search()`
     - Do NOT create or modify Qdrant collections in this module; assume caller manages collection lifecycle
