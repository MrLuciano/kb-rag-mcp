  <files>scripts/docstring-audit.py</files>

  <action>
    Extend `scripts/docstring-audit.py` with inline-comment Portuguese detection. The script already has `_has_portuguese()` and `audit_file()`. Add:

    1. **New function** `scan_inline_comments(filepath: Path) -> list[dict]`:
       - Read the file line by line
       - For each line, check if it contains a Python comment (`#`) that includes Portuguese words
       - Use the existing `_has_portuguese()` function (which checks against the `PORTUGUESE_WORDS` set)
       - Return a list of dicts: `[{"line": <int>, "text": <str>}, ...]`
       - Skip shebang lines (`#!`), encoding declarations (`# -*-`), and purely decorative lines (only `# ──` or `# ----` characters)
       - A purely decorative line = only `#`, spaces, `─`, `-`, `=`, `*` characters → no content to check

    2. **New flag** `--check-inline`:
       - When present, enables inline comment scanning in addition to docstring scanning
       - Include inline comment findings in the audit report
       - Report format for each file with inline issues:
         ```
         {relative_path}:
           Line {N}: {comment_text}
           Line {M}: {comment_text}
           -> Inline comments: {count} Portuguese
         ```

    3. **New flag** `--fail-under` (takes an integer, default None):
       - `--fail-under 0` means "exit non-zero if any Portuguese found (inline or docstring)"
       - `--fail-under N` means "exit non-zero if total findings exceed N"
       - Exit code 1 if `total_portuguese + total_inline_portuguese > fail_under`

    4. **Update main() arguments** to accept both `--check-inline` and `--fail-under`:
       - Use `argparse` (follow the pattern from `scripts/logging-audit.py` which already uses argparse)
       - Convert the existing manual `sys.argv[1:]` parsing to use argparse instead, keeping `--dir` compatibility

    5. **Update summary output** to include inline comment counts:
       ```
       Summary:
         Total public methods/classes: {N}
         OK: {N}
         MISSING: {N}
         PORTUGUESE docstrings: {N}
         PORTUGUESE inline comments: {N}
       ```

    Run the extended script against `kb_server/` and `ingest/` to confirm zero Portuguese remains after Plans 01 and 02 are complete.
  </action>

    - Script uses `argparse` with `add_argument` for `--check-inline`, `--fail-under`, `--dir`
    - Backward compatible: `python3 scripts/docstring-audit.py` (without flags) still works as before
