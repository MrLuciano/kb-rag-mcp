# Plan 16-01: Core Reclassification Engine

## Objective

Implement the core reclassification engine that detects changed classifications, backs up metadata to SQLite, updates Qdrant payloads in-place, and logs changes for audit. Provides async functions consumed by CLI commands in Plan 16-02.

## Context

From CONTEXT.md decisions:
- **D-01:** In-place metadata update (preserve embeddings)
- **D-02:** Classification fields by default (vendor, product, subsystem, doc_type, version) + optional custom fields
- **D-04:** Detect changed classifications by comparing current Qdrant metadata vs. classify() output
- **D-12:** SQLite audit table (`reclassify_history`)
- **D-13:** SQLite backup table (`reclassify_backups`) for rollback
- **D-16:** 30-day backup retention with auto-cleanup

Reusable assets:
- `ingest/classifier.py:classify()` — Run on source_file to get expected metadata
- `kb_server/vector_store.py:VectorStore` — Add `update_chunk_metadata()` method
- `ingest/registry.py:IngestRegistry` — Add schema migration for new tables

## Success Criteria

1. ✅ `VectorStore.update_chunk_metadata()` updates Qdrant payload fields without re-embedding
2. ✅ `reclassify_backups` table in registry.db stores old metadata before updates (session_timestamp, source_file, field_name, old_value)
3. ✅ `reclassify_history` table logs all changes (timestamp, source_file, field_name, old_value, new_value, session_timestamp)
4. ✅ `detect_changed_classifications()` compares Qdrant metadata vs. classify() result, returns list of changes
5. ✅ `backup_metadata()` writes to reclassify_backups before Qdrant update
6. ✅ `cleanup_old_backups()` deletes backups older than RECLASSIFY_BACKUP_RETENTION_DAYS (default 30)
7. ✅ All new functions have Google-style docstrings, English-only comments, 90%+ branch coverage
8. ✅ Test suite passes (585 baseline + new tests)

  ```python
  async def update_chunk_metadata(
      self,
      collection_name: str,
      source_file: str,
      updates: dict[str, str],
      chunk_index: int | None = None
  ) -> int:
      """
      Update metadata fields for chunks in-place without re-embedding.
      
      Args:
          collection_name: Qdrant collection name
          source_file: Source file path to filter chunks
          updates: Dict of field_name -> new_value to update
          chunk_index: Optional specific chunk index (updates single chunk)
      
      Returns:
          Number of chunks updated
      
      Raises:
          QdrantException: If Qdrant update fails
      """
      log.info(f"Updating metadata for {source_file} in {collection_name}")
      
      # Build filter
      must_conditions = [
          FieldCondition(
              key="source_file",
              match=MatchValue(value=source_file)
          )
      ]
      if chunk_index is not None:
          must_conditions.append(
              FieldCondition(
                  key="chunk_index",
                  match=MatchValue(value=chunk_index)
              )
          )
      
      filter_query = Filter(must=must_conditions)
      
      # Scroll to get point IDs
      points, _ = await self.qdrant_client.scroll(
          collection_name=collection_name,
          scroll_filter=filter_query,
          limit=10000,  # max chunks per document
          with_payload=False,
          with_vectors=False
      )
      
      if not points:
          log.warning(f"No chunks found for {source_file}")
          return 0
      
      point_ids = [p.id for p in points]
      
      # Update payload
      await self.qdrant_client.set_payload(
          collection_name=collection_name,
          payload=updates,
          points=point_ids
      )
      
      log.info(f"Updated {len(point_ids)} chunks for {source_file}")
      return len(point_ids)
  ```
