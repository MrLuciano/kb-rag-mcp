# Plan 24-03: CLI Command and Results Export

## Objective
Add a `kb-rag evaluate` CLI command that runs the full RAGAS pipeline and exports results as CSV with a console summary table.

## Tasks

### Task 1: Implement results exporter
<read_first>
- kb_server/evaluation/ragas_pipeline.py
- kb_server/evaluation/dataset.py
- observability/metrics.py (for rich console patterns)
</read_first>
<action>
Create `kb_server/evaluation/exporter.py` with:
1. `ResultsExporter` class
2. `to_csv(results: Dict, output_path: Path)` — writes CSV with columns: query, faithfulness, answer_relevancy, context_precision, context_recall, timestamp
3. `to_console(results: Dict)` — prints rich table with Metric | Mean | Min | Max | Count
4. `to_json(results: Dict, output_path: Path)` — writes detailed JSON (one object per example + summary stats)

Use `rich` library (already in requirements.txt) for console table formatting.
</action>
<acceptance_criteria>
- `ResultsExporter.to_csv()` creates file with header row and data rows
- `ResultsExporter.to_console()` prints table with 5+ columns
- `ResultsExporter.to_json()` creates valid JSON with summary.stats.mean/min/max
- Output paths use default `data/evaluation_results_YYYYMMDD_HHMMSS.csv` when not specified
</acceptance_criteria>

### Task 2: Add `kb-rag evaluate` CLI command
<read_first>
- ingest/cli/main.py (typer CLI patterns)
- ingest/cli/reclassify.py (for complex command pattern reference)
- kb_server/evaluation/ragas_pipeline.py
- kb_server/evaluation/dataset.py
- kb_server/evaluation/exporter.py
</read_first>
<action>
Add `evaluate` subcommand to `ingest/cli/main.py`:
1. `kb-rag evaluate --dataset PATH [--output PATH] [--format csv|json]`
2. Load dataset via GoldenDataset (auto-detects CSV/JSON)
3. Initialize RAGASEvaluator with current embed backend
4. Run evaluation
5. Export results (CSV + console table by default)
6. Print summary to console

Flags:
- `--dataset PATH` (required): Path to golden dataset file
- `--output PATH` (optional): Output file path (default: data/evaluation_results_YYYYMMDD_HHMMSS.csv)
- `--format FORMAT` (optional): csv or json (default: csv)
- `--backend BACKEND` (optional): Override EMBED_BACKEND for evaluation
</action>
<acceptance_criteria>
- `kb-rag evaluate --dataset test.json --output results.csv` runs without error (with mocked backend)
- `--help` shows all flags with descriptions
- Missing --dataset raises typer.BadParameter
- Invalid --format raises typer.BadParameter
- Default output path includes timestamp
</acceptance_criteria>

- Console output shows: "Evaluating with backend: {BACKEND}"
</acceptance_criteria>
