<interfaces>
Logger naming convention (established pattern from existing code):
```python
import logging
log = logging.getLogger("kb-mcp.{module}")   # for kb_server/ modules
log = logging.getLogger("kb-ingest.{module}") # for ingest/ modules
```

Existing log call patterns by level:
```python
# DEBUG — internal state, verbose internals
log.debug("Collection '%s' already exists, skipping create.", name)

# INFO — lifecycle events, user-visible operations
log.info("Collection '%s' created (dim=%d).", name, size)

# WARNING — recoverable issues, degraded behavior
log.warning("Could not create payload index '%s' on '%s': %s", field, name, exc)

# ERROR — failures, exceptions that propagate
log.error(f"Health check failed: {result}")
```

Log format (per D-06 agent discretion): use %-formatting with positional args for static messages, f-strings only for single-value template messages where the format string itself is dynamic. The observability/logging.py provides StructuredFormatter as an option but the default is stdlib formatting, which the codebase already uses.

CONTEXT.md decisions:
```
D-03: Pragmatic # pragma: no cover with inline comments allowed
D-04: Each # pragma: no cover must have inline comment explaining why
```
These are scoped to coverage, not logging — but the logging audit should note any # pragma instances it finds.

    **Design:**
    - Scan `kb_server/` and `ingest/` for all `.py` files (skip `__init__.py` and anything under `__pycache__`)
    - For each module, identify all public functions/classes/methods (names not starting with `_`)
    - Count `log.info(`, `log.debug(`, `log.warning(`, `log.error(`, `log.exception(` calls within each public function's scope
    - Report: for each module, list public functions without any log calls as GAPS
    - Output a summary table to stdout with columns: Module | Functions | Functions with logs | Coverage %

    **Output format:**
    ```
    Logging Coverage Audit
    ======================
    Generated: 2026-05-23

    kb_server/collections/router.py:
      CollectionRouter.__init__ — NO LOG
      CollectionRouter.resolve  — NO LOG
      CollectionRouter.ensure   — NO LOG
      → 0/3 public methods have log calls (0%)

    ...

    Summary:
      Total modules:  N
      Total functions: N
      Functions with logs: N  (XX%)
      Functions without logs: N
    ```

    **Naming:** Use `logging.getLogger("kb-audit")` for the script's own logger output (prints to stderr). The audit report goes to stdout.

    **No new dependencies:** Only stdlib (ast, pathlib, os, sys). No imports from the project itself — should run on any Python 3.11+ with just stdlib.

    **No CI integration:** This is a one-time audit script (per agent discretion in CONTEXT.md). The CI already has a coverage enforcement step from Plan 07-01.

    Save at `scripts/logging-audit.py`. Make it executable.
  </action>
