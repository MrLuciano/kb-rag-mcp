# Plan 20-01: Test Environment Fixes

## Objective

Fix three test environment issues: LOG_PATH PermissionError, test_reranker_lazy.py fixture isolation, and test suite cleanup.

## Tasks

### Task 1: Fix LOG_PATH PermissionError (TESTFIX-02)

**Problem:** `kb_server/server.py` uses `LOG_PATH` from `.env` which points to `/home/luck/kb-rag-mcp/logs/kb-mcp.log` — a path that doesn't exist in all environments. When tests import `kb_server.server`, `logging.basicConfig()` with `FileHandler` fails with PermissionError.

**Fix:** In `kb_server/server.py`, ensure the log directory exists before creating the FileHandler with `os.makedirs(os.path.dirname(log_path), exist_ok=True)`.

### Task 2: Fix test_reranker_lazy.py fixture isolation (TESTFIX-01)

**Problem:** Module-level `MagicMock` objects (`_mock_st`, `_mock_cross_encoder_cls`) at lines 17-19 persist across test files in the same pytest session. When run alongside other test files that import `sentence_transformers`, the migration state can cause import conflicts or test interaction.

**Fix:** Move module-level mocks into fixtures so they're scoped per-test rather than module-level.

### Task 3: Verify clean test suite (TESTFIX-03)

- Run `find . -name '*.pyc' -delete && find . -name '__pycache__' -type d -exec rm -rf {} + 2>/dev/null`
- Run full test suite with `LOG_PATH=/tmp/kb-mcp.log`
- Verify no stale artifacts or PermissionError failures

## Success Criteria
- [ ] `pytest tests/ -q --tb=short` runs without PermissionError
- [ ] `tests/test_reranker_lazy.py` passes in isolation and when run with other test files
- [ ] Clean `.pyc`/`__pycache__` artifacts removed
