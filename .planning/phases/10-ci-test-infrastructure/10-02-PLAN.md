<objective>
Replace `sys.modules` monkey-patching of `qdrant_client` with real model imports so enum comparisons work correctly without `getattr(x, 'value', x)` workarounds.

**Purpose:** Three test files (`test_smoke.py`, `test_vector_store.py`, `test_vector_store_unit.py`) inject lightweight `types.ModuleType` stubs into `sys.modules` for `qdrant_client` at module load time. These stubs create anonymous `type(name, (), {})()` instances for model classes like `Distance`, `MatchValue`, `PointStruct`, etc. When production code compares enum values (e.g., `if distance != Distance.COSINE`), the stub's anonymous-type enum returns something that doesn't match real enum values — causing incorrect comparisons. Since `qdrant-client` is already pinned in `requirements.txt` and is a lightweight pure-Python package, we can import model classes directly.

**Output:** All 3 test files import real `qdrant_client.models` classes. Full test suite passes with zero regressions. No `getattr(x, 'value', x)` workaround needed anywhere.
</objective>
