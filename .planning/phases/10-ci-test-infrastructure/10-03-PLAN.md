<objective>
Add `--fail-under` threshold flag to the logging audit script and enforce it in CI on PR-to-master to prevent logging coverage regression.

**Purpose:** The logging audit script (`scripts/logging-audit.py`) currently only reports coverage stats but doesn't have a non-zero exit code for insufficient coverage. Without CI enforcement, logging coverage can silently regress below the current baseline (~50.6%). This adds a `--fail-under PERCENT` flag and a CI step that blocks PR merges if coverage drops below the threshold.

**Output:** Updated `scripts/logging-audit.py` with `--fail-under` flag; CI workflow runs logging audit with threshold enforcement on PR-to-master.
</objective>

    1. Add `import argparse` at the top of the file (stdlib, no new dependencies)
    2. Add argument parsing in `main()`:
       ```
       parser = argparse.ArgumentParser(description="Logging coverage audit")
       parser.add_argument("--fail-under", type=float, default=None,
                           help="Exit non-zero if coverage percentage is below this threshold")
       args = parser.parse_args()
       ```
    3. After computing `total_pct`, if `args.fail_under is not None` and `total_pct < args.fail_under`:
       - Print: `FAIL: Logging coverage {total_pct:.1f}% is below threshold {args.fail_under:.1f}%`
       - `sys.exit(1)`
    4. If coverage meets or exceeds threshold, print: `PASS: Logging coverage {total_pct:.1f}% >= {args.fail_under:.1f}%`
    5. Keep the existing behavior (no args = print report, exit 0) unchanged

    Also support `PROJECT_ROOT` env var as before (used in CI to point to checkout root). The audit already reads this.

    Threshold choice: Use `40` as the initial fail_under threshold — this is below the current 50.6% baseline but above total regression. The CI step in Task 2 will use `--fail-under 40`. This gives a safety margin while accounting for the fact that utility methods are legitimately exempt.
