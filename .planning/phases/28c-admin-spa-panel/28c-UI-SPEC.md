## Interaction Contracts

### Auth Flow (SPA-02, SPA-03, SPA-12)

1. **Initial load:** Alpine.js `init()` checks `localStorage` for saved API key
2. **Key found:** Sets `Authorization: Bearer` header on all HTMX requests via `htmx.config.headers`; attempts silent auth via `/api/v1/auth/session`
3. **Key not found or auth fails:** Shows login modal overlay (full-page backdrop, centered card)
4. **User submits key:** POST `/api/v1/auth/session` with Bearer token
   - **Success (200):** Receives HttpOnly JWT session cookie (8h, SameSite=Lax). Stores API key in `localStorage`. Sets `isAuthenticated = true`. Closes modal. Triggers HTMX tab load.
   - **Failure (401/403):** Shows error message in modal. Clears `localStorage`. Stays in login state.
5. **401 interceptor:** HTMX global event handler catches 401 responses ŌåÆ shows login modal
6. **Logout:** Clears `localStorage`, resets Alpine state, shows login modal

### Tab Navigation (SPA-04, SPA-05)

1. **Tab click:** Alpine.js sets `activeTab = tabName`. HTMX `hx-get="/admin/tabs/{name}"` loads content
2. **Active tab:** Sidebar item gets `bg-primary` active class, inactive items show `text-light` on hover
3. **Tab content:** HTMX swaps inner HTML of `#tab-content` div
4. **Tab partials** are server-rendered Jinja2 HTML fragments (not JSON)
5. **Role gating:** Admin tab (`x-show="!tab.admin_only || userRole === 'admin'"`) hidden from non-admin roles
6. **Browser navigation:** URL hash updates on tab switch for history support

### Monitor Lights Bar (SPA-06)

1. **Auto-refresh:** `hx-trigger="every 30s"` on the monitor-lights container div
2. **Data source:** `GET /admin/tabs/monitor-lights` endpoint calls `check_all_components()`
3. **Visual:**
   - **Green dot** (`badge bg-success`, 12px circle): Component healthy
   - **Yellow dot** (`badge bg-warning`): Component degraded/warning
   - **Red dot** (`badge bg-danger`): Component unhealthy/error
   - **Gray dot** (`badge bg-secondary`): Component not configured
4. **Interaction:**
   - Component name label below dot
   - Latency displayed in ms below name
   - "details" toggle expands Alpine.js `x-show` with component data dict
   - Click to expand/collapse per component

### Config Inline Editing (SPA-08)

1. **View:** Config table shows Group badges, Key (code), Value, Type badges, Actions
2. **Edit:** Double-click value cell ŌåÆ Alpine.js switches from `x-text` to `<input>` with `x-model`
3. **Save:** On blur or Enter key, HTMX PUT to `/api/v1/config/{key}` with new value
4. **Cancel:** Escape key or click outside restores original value
5. **Search:** `input` above table filters rows by key name via Alpine.js `x-model` + `x-show`
6. **Reset All:** Button with `hx-confirm` ŌåÆ POST `/api/v1/config/reset` reloads table

### Document Browse Cleanup (SPA-09)

1. **Selection:** Checkbox column with "select all" header checkbox
2. **Toolbar:** Appears when Ōēź1 checkbox checked ŌĆö shows "Delete" and "Re-ingest" and "Delete Failed" buttons
3. **Per-document:** Actions dropdown (View, Delete with `hx-confirm`, Re-ingest)
4. **Confirmation:** `hx-confirm` for delete: "Delete this document? This action cannot be undone."

### Profile Tab (SPA-07)

1. **Account info:** Username, role badge, created date displayed via Alpine.js `x-text`
2. **API keys table:** Prefix, description, created, status badge (Active/Revoked), Revoke button
3. **Generate New Key:** POST to `/api/v1/api-keys`, show raw key once in success alert with "Copy" button
4. **GDPR Export:** Download JSON via blob + anchor click pattern
5. **Erasure:** Confirm dialog ŌåÆ POST ŌåÆ show status

## Responsive Behavior

> BootStrap 5 grid is inherently responsive. Admin panel is **desktop-first** with basic mobile fallback.

| Breakpoint | Layout |
|------------|--------|
| Ōēź992px (lg) | Full sidebar (280px) + tab content side by side |
| 768-991px (md) | Collapsed sidebar (icon-only, 60px) + tab content fills remaining width |
| <768px (sm) | Sidebar hidden behind hamburger toggle; full-width tab content |
| Monitor lights | `d-flex flex-wrap` collapses to 2-3 columns on narrower viewports |

**Mobile fallback approach:**
- Sidebar collapses: Icons without labels, hamburger toggle for overlay expansion
- Tables: Bootstrap `.table-responsive` wrapper for horizontal scroll
- Forms: Stack inputs vertically on small screens
- Modals: Use `modal-fullscreen-sm-down` for login on very small screens
