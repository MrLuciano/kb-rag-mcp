       - `test_batch_processor_end_to_end` — likely needs full stack → tag

       - If any test creates a real ingestion run (calling `ingest.ingest.main()`),
         tag that specific test with `@pytest.mark.integration`.
       - If all tests run with mocked dependencies, do NOT add markers.

    3. **tests/test_worker_system.py** — Tests rate limiter, file worker, pool.
       - Rate limiter and pool tests use mocks → do NOT tag
       - If `test_executor_submits_to_pool` or similar tests reach real Qdrant → tag
       Use per-function markers if mixed.

    4. **tests/test_validation.py** — Tests validation pipeline.
       Validation pipelines are typically pure Python (format/size/content checks).
       If any test requires Qdrant or live services → tag individually.
       Most should be pure unit tests → no marker needed.

    5. **tests/test_job_system.py** — Tests job scheduling.
       - Scheduler tests use mock time/events → do NOT tag
       - If any test triggers actual ingest execution with Qdrant upsert → tag

    6. **tests/test_file_watcher.py** — Tests file system monitoring.
       Filesystem operations don't need Qdrant/LM Studio → no markers.
       (Unless a test triggers ingestion, which should be tagged.)

    **Tagging rules:**
    - Add `import pytest` at the top if not present
    - Add `@pytest.mark.integration` decorator on individual test functions
      that need live infra
    - **Do NOT** add module-level `pytestmark` unless ALL tests in the file
      require live services
    - **Do NOT** modify test logic — only add markers
    - **Do NOT** tag tests that work with `unittest.mock.patch`, `AsyncMock`,
      or `MagicMock` (they're properly isolated)
