  <name>Task 1: Add per_source_summary method to IngestRegistry</name>
  <files>ingest/core/metadata.py</files>
  <action>
    Add a public method `per_source_summary(self) -> list[dict]` to the `IngestRegistry` class.
    Place it after the existing `summary()` method (after its log_reg.info line near line 473).

    The method queries the `files` table grouped by source directory. Source directory is the
    first path component before `/` in the `path` column. Files at root (no `/` in path) are
    grouped under `"(root)"`.

    SQL to use — parameterized (no string interpolation):
    ```sql
    SELECT
        CASE
            WHEN INSTR(path, '/') > 0
            THEN SUBSTR(path, 1, INSTR(path, '/') - 1)
            ELSE '(root)'
        END AS source,
        COUNT(*)                                          AS files,
        SUM(CASE WHEN status = 'ok' THEN 1 ELSE 0 END)    AS ok,
        SUM(CASE WHEN status = 'error' THEN 1 ELSE 0 END) AS errors,
        COALESCE(SUM(chunks), 0)                          AS chunks,
        MAX(indexed_at)                                   AS last_indexed
    FROM files
    GROUP BY source
    ORDER BY source
    ```

    Return a list of dicts, each with keys: `source` (str), `files` (int),
    `ok` (int), `errors` (int), `chunks` (int), `last_indexed` (float | None).

    Log result count: `log_reg.debug("Per-source summary: %d sources", len(results))`

    Do NOT modify any existing methods. Do NOT change schema.
  </action>
  <verify>
