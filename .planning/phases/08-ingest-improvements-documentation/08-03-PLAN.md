    """Audit a single Python file for logging coverage on public functions/classes."""
    with open(filepath) as f:
        tree = ast.parse(f.read(), filename=str(filepath))

    for node in ast.walk(tree):
        if isinstance(node, (ast.FunctionDef, ast.AsyncFunctionDef)):
            if node.name.startswith("_"):
                continue
            # ... check for body statements ...

    Create `scripts/docstring-audit.py` — an AST-based scanner that identifies public
    functions, methods, and classes in Python modules without proper English Google-style
    docstrings. Adapt the pattern from `scripts/logging-audit.py`.

    The script must:
    1. Accept a list of source directories (default: `kb_server/` and `ingest/`)
    2. Walk all `.py` files (skip `__init__.py`, `__pycache__`)
    3. For each file, parse with `ast.parse()` and walk the AST
    4. Find all `FunctionDef`, `AsyncFunctionDef`, and `ClassDef` nodes that are public
       (don't start with `_`)
    5. For each, check if the first body statement is an `Expr` containing a `Constant` string
       (i.e., a docstring)
    6. If no docstring exists, record as "MISSING"
    7. If docstring exists, check for Portuguese indicators: words like "classe", "função",
       "retorna", "args", "paramêtro", "arquivo", "diretório", "usuário", "configura",
       "extrai", "infere", "gerencia", "utiliza" (heuristic, not exhaustive)
    8. If Portuguese detected, record as "PORTUGUESE"
    9. Also check that Google-style docstrings have Args/Returns/Raises sections where
       the function has parameters / returns non-None / is documented with sections
    10. Output format — print to stdout:
        ```
        Docstring Coverage Audit
        ======================
        Generated: {date}

        {relative_path}:
          {function_name}: {MISSING|PORTUGUESE|OK}
          ...

        Summary: {X}/{Y} functions with docstrings | {Z} Portuguese | {W} missing
        ```
    11. Exit code: 0 always (informational, not a gate)
    12. Accept `--dir` option to scan specific directories

    Use the same `log_vars` / `log_methods` style as logging-audit.py — define the check
    criteria at the top of the file for easy maintenance.

    Portuguese word list for detection (`PORTUGUESE_WORDS`):
    `["classe", "função", "retorna", "args", "parâmetro", "parâmetros", "arquivo",
    "diretório", "usuário", "configura", "extrai", "infere", "gerencia", "utiliza",
    "classificador", "documento", "nome", "caminho", "pasta"]`
