### CSP Policy

```python
CSP_DIRECTIVES = (
    "default-src 'self'; "
    "script-src 'self' 'nonce-{nonce}' "
    "https://cdn.jsdelivr.net https://unpkg.com; "
    "style-src 'self' 'unsafe-inline' "
    "https://cdn.jsdelivr.net; "
    "frame-src 'self' https:; "
    "object-src 'none'"
)
```

Nonce generated per-request via `secrets.token_hex(16)` in CSPMiddleware (`app.py:35`). All inline scripts use `nonce="{{ get_nonce(request) }}"`.

### Auth Flow Summary

1. User visits `/admin` → `shell.html` loads with Alpine.js overlay (`x-show="!isAuthenticated"`)
2. User enters API key → frontend POSTs to `/api/v1/auth/session` with `Authorization: Bearer {key}`
3. Server validates key, sets `HttpOnly` session cookie with HMAC signature
4. Subsequent requests use session cookie (validated in `get_current_user` via HMAC + expiry + DB revocation check)
5. 401 responses from HTMX trigger `show-login` event → login overlay reappears
