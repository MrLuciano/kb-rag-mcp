<objective>
Delete the legacy `server/` module and verify the codebase exclusively uses `kb_server/`.

Purpose: `server/` is a stale duplicate of `kb_server/`. Keeping two modules creates confusion, import ambiguity, and maintenance burden. Deleting it enforces the single canonical module.
Output: `server/` directory removed; grep confirms zero external imports.
</objective>

<task type="auto">
  <name>Task 1: Audit external imports of server/ then delete the directory</name>
  <files>server/ (deleted)</files>
  <read_first>
    - Run: `grep -rn "from server\." --include="*.py" . | grep -v "^./server/" | grep -v ".venv" | grep -v __pycache__ | grep -v .worktrees`
    - Run: `grep -rn "import server" --include="*.py" . | grep -v "^./server/" | grep -v ".venv" | grep -v __pycache__ | grep -v .worktrees`
  </read_first>
  <action>
1. Run the two grep commands above. Collect every file outside `server/` that imports from the `server` package.
2. For each such file: update the import to the equivalent `kb_server` path. The mapping is 1-to-1 — `server/` and `kb_server/` share the same structure:
   - `from server.cache.manager import CacheManager` → `from kb_server.cache.manager import CacheManager`
   - `from server.embed_client import ...` → `from kb_server.embed_client import ...`
   - `from server.vector_store import VectorStore` → `from kb_server.vector_store import VectorStore`
   - `from server.health import ...` → `from kb_server.health import ...`
3. After updating all external imports, delete the entire `server/` directory:
   ```
   rm -rf server/
   ```
4. Verify deletion: `ls server/` should return "No such file or directory".
  </action>
  <verify>
    <automated>grep -rn "from server\." --include="*.py" . | grep -v ".venv" | grep -v __pycache__ | grep -v .worktrees | grep -c "." || echo "0 external server imports remain"</automated>
  </verify>
  <done>
    - `server/` directory is gone (`ls server/` → No such file)
    - `grep -rn "from server\." . | grep -v ".venv" | grep -v __pycache__ | grep -v .worktrees` returns zero results
  </done>
</task>
