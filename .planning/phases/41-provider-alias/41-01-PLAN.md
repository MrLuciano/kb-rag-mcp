<objective>
**What:** Configurable provider name aliases for the multi-backend embedding system — admin defines aliases via config entries (group `provider_alias`, key pattern `provider_alias.<canonical_name>`), and EmbedClient resolves them lazily at provider-selection time with hot-reload support.

**Purpose:** Teams running multiple embedding backends can assign short memorable names (e.g., `aliyun` → `dashscope`, `my-local` → `lmstudio-rest`) without modifying the `EMBED_BACKEND` env var or provider chain. The mapping is live-editable via the config API and takes effect without server restart.

**Output:**
- `ConfigLoader.get_aliases()` — enumerate all alias mappings as `dict[str, str]`
- `ConfigLoader.resolve_alias(alias_name)` — resolve single alias, returns `Optional[str]`
- `ConfigLoader._notify_observers` enhanced to support `prefix.*` wildcard patterns
- `embed_client._resolve_alias(provider)` — lazy per-call alias resolution
- `embed_client.init_alias_resolution(loader)` — hot-reload hook registration
- `embed_client.validate_providers` — updated to accept aliased provider names
- `tests/test_provider_alias.py` — comprehensive test file
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-01 | Tampering | config table `provider_alias.*` entries | mitigate | Alias values are validated at write time by ConfigLoader.set() (type validation). EmbedClient falls back to value-as-is when alias resolves to an unknown provider name — unresolved aliases log a warning and do not block the request. |
| T-41-02 | Spoofing | EmbedClient `_try_provider` alias lookup | mitigate | Alias resolution only maps names, not URLs or API keys (D-04). An alias pointing to `"dashscope"` still requires `dashscope` to be registered in `_BACKENDS` — the circuit breaker and budget layer provide defense-in-depth. |
| T-41-03 | Information Disclosure | wildcard observer logging | accept | `_on_alias_changed` logs alias keys at debug level only (`log.debug`). No PII or secrets in alias values (D-04 restricts to name → name mapping). |
| T-41-SC | Tampering | pip installs | mitigate | No new dependencies in this phase. All existing package installations were vetted in prior phases. No slopcheck needed. |
</threat_model>
