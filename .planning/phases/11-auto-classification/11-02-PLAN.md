<objective>
Add document metadata extraction (PDF/DOCX title, author, subject, keywords) and gap-filling enrichment to the classifier, then integrate vendor and subsystem into the ingest pipeline's chunk metadata.

**Purpose:** When filename patterns are ambiguous, document metadata (title, subject, author, keywords from PDF/DOCX) provides additional signal for classification. This fills gaps without LLM dependency. The same pattern-matching logic already used for filenames runs against extracted metadata text, giving better classification for documents with unclear filenames. Also integrates the new vendor and subsystem fields from Plan 01 into the ingest pipeline so they're stored in Qdrant chunk payloads.

**Output:** Updated `ingest/classifier.py` with metadata extraction and enrichment; updated `ingest/ingest.py` and `ingest/worker/batch_processor.py` to store vendor/subsystem in chunk metadata; corresponding tests.
</objective>

       ```python
           PDF/DOCX metadata (title, subject, author, keywords) and reruns the

           1. _meta.json overrides (already applied before this function)
           2. Filename/directory heuristics (current classification)
           3. Document metadata heuristics (this function — gap-fill only)
           
           Args:

               docs_root: Root documents directory.
               current: Current classification dict from classify().
           
           Returns:
               Updated classification dict (new copy, original unchanged).
           """
       ```

       Implementation:
       - Create a new dict copy of `current`
       - Define which values are "defaults" that indicate gaps:
         - vendor: "" (empty)
         - product: "geral"

         ```python
         class _MetadataPath:
             """Wrapper to make metadata text look like a Path for reuse of inference functions."""
             def __init__(self, text: str):
                 self._text = text
             
             @property
             def stem(self):
                 return self._text
             
             @property
             def suffix(self):
                 return ""
             
             @property
             def name(self):
                 return self._text
         ```

       - Better approach: pass metadata text directly to the inference logic.

       OR — even simpler approach: use a `Path` wrapper that stores the text as the stem and has empty suffix:
       ```python
       class _MetadataPath(Path):
           """Path subclass that exposes metadata text as the filename stem."""
           def __new__(cls, text: str):
               return super().__new__(cls, text)
           
           @property
           def stem(self):
               return self._text if hasattr(self, '_text') else super().stem
       ```

       ```python
       # that feeds metadata text through the same inference functions.
       # Only stem and suffix are used by the inference functions.
       class _ClassificationText:
           """Simulate Path fields for running classifiers on arbitrary text."""

               self.suffix = ""
               self.name = text
               self.parent = Path(".")  # Minimal parent so relative_to doesn't crash
       ```

       Actually wait — `Path("OpenText WebReport Administrator Guide")` would work as a stem for all inference functions. The `.suffix.lower()` check in `infer_doc_type` for artifact types would return "" which doesn't match any artifact suffix, so that's fine.

       Let me keep it simple — just use `Path(text)` directly:

       ```python
       def _classify_from_metadata_text(
           metadata_text: str,
           docs_root: Path,
           product_override: str | None = None,
       ) -> dict[str, str]:
           """Run classification heuristics on a metadata text string."""
           if not metadata_text.strip():
               return {}
           
           # Create a synthetic Path so we can reuse existing inference functions
           synthetic = Path(metadata_text)
           
           return {
               "vendor": infer_vendor(synthetic, product_override or ""),
               "product": infer_product(synthetic, docs_root, product_override),
               "doc_type": infer_doc_type(synthetic),
           }
       ```

       Then in `enrich_classification()`:
       ```python
       result = dict(current)
       
       # Determine which fields need gap-filling
       defaults = {"", "geral", "document"}
       gaps = {
           k for k, v in result.items()
           if v in defaults or not v  # catches "" and empty
       }
       
       if not gaps:
           return result
       
       metadata = extract_document_metadata(file_path)
       if not metadata or not any(metadata.values()):
           return result
       
       metadata_text = _build_metadata_text(metadata)
       if not metadata_text.strip():
           return result
       
       # Run classification on metadata text
       meta_classification = _classify_from_metadata_text(
           metadata_text, docs_root, result.get("product")
       )
       
       # Only fill gaps — never override non-default values
       if result.get("vendor") in defaults and meta_classification.get("vendor"):
           result["vendor"] = meta_classification["vendor"]
       if result.get("product") in defaults and meta_classification.get("product"):
           result["product"] = meta_classification["product"]
       if result.get("doc_type") in defaults and meta_classification.get("doc_type"):
           result["doc_type"] = meta_classification["doc_type"]
       
       return result
       ```
