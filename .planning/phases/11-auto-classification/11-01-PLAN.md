<objective>
Add vendor and subsystem inference to the document classifier, extending the existing product/doc_type inference with new classification dimensions.

**Purpose:** Documents need to be classified by vendor (e.g., OpenText) and subsystem (e.g., a sub-product area) in addition to the existing product, doc_type, and version fields. This enables more precise metadata filtering and retrieval. Vendor is inferred from filename patterns (e.g., "OpenText") and product-to-vendor mapping. Subsystem is inferred from directory hierarchy below the product level.

**Output:** Updated `ingest/classifier.py` with `infer_vendor()`, `infer_subsystem()`, their pattern tables, and updated `classify()`/`classify_document()` returning vendor and subsystem. Corresponding tests.
</objective>

    1. Add `SUBSYSTEM_PATTERNS: list[tuple[str, list[str]]]` — optional pattern-based subsystem detection from filenames:

       - ("Security", [r"\bsecurity\b", r"\bauth\b", r"\bpermission\b", r"\bacl\b"])

    2. Add `infer_subsystem(file_path: Path, docs_root: Path) -> str`:
