# Phase 54-04: UX Feature Additions

**Goal:** Add dismissible alerts, RAGAS progress indication, and search pagination to improve the user experience.

## Tasks

### Task 1: Add dismiss buttons to alert banners

**Files:** `kb_server/ui/templates/base.html` (HTMX error handlers)

The HTMX error handlers in `base.html` inject alerts inline without a close button. Add the Bootstrap `alert-dismissible` pattern with a close button to dynamically created alerts.

**Implementation approach:**
1. In the HTMX `htmx:responseError` handler, add a `<button type="button" class="btn-close" data-bs-dismiss="alert">` inside the injected alert div.
2. Add the `alert-dismissible` class to the alert div.
3. Do the same for the `htmx:sendError` network error handler.

**Verification:** Dynamically injected error alerts have a close button; clicking it dismisses the alert.

### Task 2: Add progress indication for RAGAS evaluation runs

**File:** `kb_server/ui/templates/admin/tab_ragas.html:12-19`

The RAGAS evaluation button shows a spinner only during the initial HTMX load. Once evaluation starts, there is no progress feedback. Add a progress bar or animated status indicator that shows during evaluation execution.

**Implementation approach:**
1. After the evaluation button is clicked, show a Bootstrap progress bar (`<div class="progress-bar progress-bar-striped progress-bar-animated">`) indicating "Running evaluation..."
2. Replace with results when done.

**Verification:** Running a RAGAS evaluation shows visible progress feedback beyond the button spinner.

### Task 3: Add pagination to search results page

**File:** `kb_server/ui/templates/search_results.html:6-26`

Search results display all results in a single scrollable list with no pagination. Add Bootstrap pagination controls below the results to allow navigating through pages.

**Implementation approach:**
1. Add page navigation controls at the bottom of the search results list (following the pattern in `browse.html` or `_documents_table.html`)
2. Wire up page links to the search query with page parameter

**Verification:** Search results with more than N items show pagination controls; clicking them navigates between pages.
