# Phase 54-03: Layout & Spacing Fixes

**Goal:** Fix spacing issues, container nesting, and layout alignment across templates.

## Tasks

### Task 1: Remove double container nesting on error page

**File:** `kb_server/ui/templates/error.html:6`

The error template inherits from `base.html` which already provides a `<div class="container">`. The error template then wraps its content in another `<div class="container">`, creating double horizontal padding. Remove the inner `.container` div while preserving the grid structure.

**Before:**
```html
{% extends "base.html" %}
{% block content %}
<div class="container">
    <div class="alert alert-danger" role="alert">
```

**After:**
```html
{% extends "base.html" %}
{% block content %}
<div class="alert alert-danger" role="alert">
```

**Verification:** `grep '<div class="container">' kb_server/ui/templates/error.html` returns 0 matches (the base template provides it).

### Task 2: Clean up pagination href whitespace

**File:** `kb_server/ui/templates/browse.html:138-146`

The pagination Previous/Next buttons have Jinja2 whitespace in the `href` attribute which can produce URLs with embedded newlines and spaces. Use Jinja2's `-` strip markers or reconstruct the href to avoid this.

**Verification:** Pagination href attributes contain no whitespace/newline characters.

### Task 3: Center job status counters in admin card

**File:** `kb_server/ui/templates/admin/_job_status.html:4`

The job status counters (Completed/Pending/Failed) are left-aligned in a wide card. Center them using Bootstrap flex utilities: add `justify-content-center` or `mx-auto` to the flex container.

**Verification:** Job status counters are visually centered in the card.

### Task 4: Add spacing on mobile search results area

**File:** `kb_server/ui/templates/search.html:86`

The mobile search results area stacks the info alert flush against the card bottom with no bottom margin. Add `mb-3` to the `<div id="results">` container.

**Verification:** Results area has spacing below on mobile.
