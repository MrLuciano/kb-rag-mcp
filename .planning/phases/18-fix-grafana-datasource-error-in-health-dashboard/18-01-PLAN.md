# Plan 18-01: Fix Grafana Datasource Error

## Objective

Fix the `Datasource ${DS_PROMETHEUS} was not found` error in the Grafana health dashboard by replacing template variable references with a stable datasource UID across both Docker Compose and Helm deployment paths.

## Tasks

### Task 1: Add UID to Docker Compose datasource provisioning

- Edit `deployment/config/grafana-provisioning/datasources/prometheus.yml`
- Add `uid: prometheus` after line 8 (`name: Prometheus`)
- This matches D-04 — both datasource configs need explicit UIDs

### Task 2: Fix Docker Compose dashboard JSON

- Edit `deployment/config/grafana-dashboard.json`
- Replace all 63 occurrences of `"${DS_PROMETHEUS}"` with `"prometheus"`
- Remove the `__inputs` section (lines 2-10) per D-02
- `__requires` section can remain (no functional impact per D-01 discretion)

### Task 3: Add UID to Helm datasource provisioning

- Edit `deployment/helm/kb-rag-mcp/templates/configmap-monitoring.yaml`
- Add `uid: prometheus` after `name: Prometheus` in the datasource ConfigMap block (line 57)
- Matches D-04 for the Helm deployment path

### Task 4: Fix Helm dashboard JSON

- Edit `deployment/helm/kb-rag-mcp/dashboards/grafana-dashboard.json`
- Same changes as Task 2: replace 63 `"${DS_PROMETHEUS}"` with `"prometheus"`, remove `__inputs` section

## Success Criteria

- [ ] Task 1: Docker Compose datasource YAML has `uid: prometheus`
- [ ] Task 2: Docker Compose dashboard JSON has zero `DS_PROMETHEUS` references, no `__inputs`
- [ ] Task 3: Helm datasource ConfigMap has `uid: prometheus`
- [ ] Task 4: Helm dashboard JSON has zero `DS_PROMETHEUS` references, no `__inputs`
