    **Key insight from code analysis:** `kb_server/server.py` does NOT import `starlette`, `fastapi`, `uvicorn`, `sentence_transformers`, or `fastembed` at module level (all lazy imports are inside `_search_kb`). The `mcp` and `qdrant_client` packages are real installed packages. The conftest's session-scoped `mock_qdrant_client` (autouse) patches `AsyncQdrantClient` for all tests. Therefore, **no module stubs are needed at all** for `importlib.import_module("kb_server.server")` to succeed. The `_ensure_stubs()` function is entirely vestigial.

    ### Changes to tests/test_smoke.py:

    1. **Delete `_ensure_stubs()` function** (lines 17-150) and its module-level call on line 151. Remove `import types` (no longer needed). Keep `import sys` (still used for sys.path in the fixture).

    2. **Delete the module-level `sys.path.insert(0, ...)` on line 152.** This adds a non-existent `server/` directory (the legacy `server/` package was deleted in Phase 1). Pytest already adds the project root to sys.path, so `kb_server.*`, `observability.*`, `config.*` and all other project packages are importable.

    3. **Add a per-function fixture** before the `_run()` helper:

    ```python
    @pytest.fixture
    def patch_smoke_deps():
        """Per-function fixture: enables kb_server.server import without module-level
        sys.modules pollution. Unlike the old _ensure_stubs(), this does NOT stub
        starlette.*, fastapi, or mcp.server.sse — those modules must remain real
        so test_sse_handler.py can run in the same process (per D-01, D-02).

        The fixture is intentionally minimal — all packages imported by server.py
        (mcp, qdrant_client, httpx) are real installed packages, and our own
        modules (kb_server.*) import without side effects.
        """
        sys.path.insert(
            0, os.path.join(os.path.dirname(__file__), "..", "server")
        )
        yield
        # sys.path teardown is automatic on fixture scope exit
    ```

    4. **Add `patch_smoke_deps` parameter** to all three test functions:

    ```python
    @pytest.mark.integration
    def test_search_kb_returns_text_content_on_results(monkeypatch, patch_smoke_deps):
    ```

    ```python
    @pytest.mark.integration

    ```

    ```python

    ```

    5. **Remove unused imports**: Remove `import types` (line 12). Keep `import sys`, `import os`, `import asyncio`.

    - Do NOT modify any other test files

    ### Why this works:

    - `server.py` imports `from mcp.server.sse import SseServerTransport` — this resolves to the REAL installed `mcp` package (mcp==1.27.1), triggering starlette imports internally, which also succeed (starlette==1.0.0)

    - VectorStore.__init__() calls `get_embed_dim()` which is a pure dict lookup in `embed_client.py` — no network I/O
    - No module-level sys.modules stubs remain, so test_sse_handler.py can import real starlette regardless of test collection order
