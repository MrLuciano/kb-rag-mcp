## Quick Task: Fix Missing SQLAlchemy Dependency

### Finding

SQLAlchemy is **already present** in both dependency files:
- `requirements.in` line 39: `SQLAlchemy>=2.0.0`
- `requirements.txt` line: `sqlalchemy==2.0.51`

No code changes were required.

### Root Cause (Corrected)

The Docker container restart loop observed during Phase 40 UAT was **not** caused by a missing dependency in `requirements.txt`. The dependency was added in a prior commit but the Docker image was never rebuilt.

### Resolution

Rebuild the Docker image and restart the container:

```bash
docker compose build --no-cache
docker compose up -d
```

Or, if using plain Docker:

```bash
docker build -t kb-rag-mcp:latest .
docker run -d --name kb-rag-mcp kb-rag-mcp:latest
```

### Verification After Rebuild

```bash
docker exec kb-rag-mcp python -c "import sqlalchemy; print(sqlalchemy.__version__)"
```

Expected output: `2.0.51`

### UAT Note

The Phase 40 UAT diagnosis has been corrected. Test 6 (Server Integration) was previously marked as "blocked" due to a pre-existing SQLAlchemy issue. That issue is resolved by rebuilding the Docker image — no code fix was needed.
