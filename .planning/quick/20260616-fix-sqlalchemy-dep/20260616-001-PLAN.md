## Objective

Add the missing SQLAlchemy dependency to requirements.in and regenerate requirements.txt with pip-compile so the Docker image includes SQLAlchemy and the MCP server boots correctly.

## Background

During Phase 40 UAT, the Docker container was found to crash on startup with:
```
ModuleNotFoundError: No module named 'sqlalchemy'
```

`sqlalchemy` is installed in `.venv` (2.0.49) but missing from `requirements.in` and `requirements.txt`. The auth module (`kb_server/auth/models.py`) imports SQLAlchemy.

## Tasks

### T1: Add SQLAlchemy to requirements.in
- Read `requirements.in`
- Add `SQLAlchemy==2.0.49` to the file
- Commit: `chore(deps): add SQLAlchemy==2.0.49 to requirements.in`

### T2: Regenerate requirements.txt
- Run `pip-compile` to regenerate `requirements.txt` from `requirements.in`
- Verify SQLAlchemy appears in the generated `requirements.txt`
- Commit: `chore(deps): regenerate requirements.txt with SQLAlchemy`

### T3: Verify fix
- Run `pip install -r requirements.txt --dry-run` or inspect `requirements.txt`
- Confirm SQLAlchemy and its transitive deps are listed
- Confirm no duplicate/conflicting versions

## Verification
- `grep -i sqlalchemy requirements.txt` returns `SQLAlchemy==2.0.49`
- `pip-compile` exits 0
