# Plan 24-01: RAGAS Core Evaluator (Custom Metrics)

## Objective
Implement the core RAGAS-style evaluation pipeline with 4 metrics (faithfulness, answer_relevancy, context_precision, context_recall) using our existing `llm_wrapper.py` LLM-as-judge backend (LM Studio/Ollama/OpenAI-compat).

**Design decision:** Instead of importing the `ragas` library (which has incompatible transitive dependencies on langchain-community 0.3.x), we implement the 4 metrics directly using LLM prompts. Each metric sends 1-2 prompts to the judge LLM and parses a 0-1 score. This gives us full control, zero new dependencies, and identical evaluation quality.

## Tasks

### Task 1: Implement custom metrics module
<read_first>
- kb_server/evaluation/llm_wrapper.py
- kb_server/evaluation/dataset.py
</read_first>
<action>
Create `kb_server/evaluation/metrics.py` with 4 async metric functions:

1. `faithfulness(answer: str, contexts: list[str], llm) -> float`
   - Prompt: "Is the answer fully supported by the context? Rate 0-1."
   - Parse float from LLM response

2. `answer_relevancy(question: str, answer: str, llm) -> float`
   - Prompt: "Does the answer directly address the question? Rate 0-1."
   - Parse float from LLM response

3. `context_precision(question: str, contexts: list[str], llm) -> float`
   - Prompt: "What fraction of the provided contexts are relevant to the question? Rate 0-1."
   - Parse float from LLM response

4. `context_recall(question: str, answer: str, ground_truth: str, contexts: list[str], llm) -> float`
   - Prompt: "What fraction of facts in the ground_truth answer are present in the contexts? Rate 0-1."
   - Parse float from LLM response

Add a `_parse_score(text: str) -> float` helper that extracts a 0-1 float from free-form LLM output (regex for decimal, or keywords like "yes"/"no" mapped to 1.0/0.0).
</action>
<acceptance_criteria>
- All 4 metric functions are async and return float
- `_parse_score` handles "0.85", "85%", "yes", "no", "high", "low"
- Functions accept the `llm` parameter (any object with `.invoke(prompt)` coroutine)
</acceptance_criteria>
