# Plan 24-04: Langchain LLM Wrapper for RAGAS

## Objective
Create a Langchain-compatible LLM wrapper that adapts our existing text-generation backends (LM Studio, Ollama, OpenAI-compat) to RAGAS's expected `BaseLanguageModel` interface. This wrapper is required for `RAGASEvaluator.evaluate()` to function — RAGAS metrics call LLM-as-judge internally.

## Background

RAGAS `evaluate()` accepts a `llm=` parameter that must implement Langchain's `BaseLanguageModel` or `BaseChatModel` interface (at minimum: `invoke()` or `generate()` methods returning `AIMessage` or `LLMResult`). Our project currently has:

- `kb_server/embed_client.py` — embedding backends (vectors only, no text generation)
- Direct HTTP clients (httpx) for LM Studio REST, Ollama, and OpenAI-compat endpoints
- No existing Langchain integration

The wrapper must bridge our HTTP-based text generation to Langchain's expected interface.

### Task 2: Implement BaseLLMWrapper abstract base
<read_first>
- tests/test_embed_client_unit.py (for httpx mocking patterns)
- kb_server/embed_client.py (for config reading patterns)
</read_first>
<action>
Create `kb_server/evaluation/llm_wrapper.py` with:

1. `BaseLLMWrapper(ABC)` abstract class with:
   - `invoke(prompt: str, **kwargs) -> str` — takes a prompt string, returns generated text string
   - `invoke_batch(prompts: List[str], **kwargs) -> List[str]` — batch generation
   - `model_name: str` property

2. Four concrete implementations:
   - `LMStudioRestWrapper` — wraps LM Studio REST API (`/v1/chat/completions`)
   - `OpenAICompatWrapper` — wraps generic OpenAI-compatible API
   - `OllamaWrapper` — wraps Ollama generate/chat API
   - `LMStudioSDKWrapper` — wraps LM Studio Python SDK (if available; gracefully degrade if SDK not installed)

3. Factory function `create_llm_wrapper(backend: str, **kwargs) -> BaseLLMWrapper`
   - Reads `LMS_BASE_URL`, `OLLAMA_HOST`, `OPENAI_API_KEY`, `OPENAI_BASE_URL` from environment
   - Returns appropriate wrapper instance
   - Raises `ValueError` for unsupported backend

4. Langchain adapter `RAGASLLMAdapter(BaseLanguageModel)` or `SimpleLLM(BaseLanguageModel)`:
   - Implements Langchain's `invoke()` method that calls `BaseLLMWrapper.invoke()`
   - Wraps the string response in `AIMessage`
   - This is the object passed to `ragas.evaluate(llm=...)`
</action>
<acceptance_criteria>
- `create_llm_wrapper("lmstudio-rest")` returns LMStudioRestWrapper instance
- `create_llm_wrapper("ollama")` returns OllamaWrapper instance
- `create_llm_wrapper("unsupported")` raises ValueError
- All wrappers implement `invoke()` returning str
- Langchain adapter implements `invoke()` returning AIMessage
- No live services needed for instantiation (lazy connection)
</acceptance_criteria>
