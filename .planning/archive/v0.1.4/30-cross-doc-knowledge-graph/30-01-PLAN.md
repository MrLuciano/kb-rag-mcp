<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ingested document content → graph builder | Untrusted content influences derived entities/topics/relationships |
| graph metadata → vector store payload | Derived relationship data becomes queryable server state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-01 | Tampering | graph metadata derivation | mitigate | Keep derivation deterministic and bounded; avoid executing or importing user-provided content |
| T-30-02 | Information Disclosure | cross-KB graph linkage | mitigate | Scope graph identifiers and lookups to a single collection/KB |
| T-30-03 | Denial of Service | oversized graph metadata | accept | Use bounded, lightweight graph fields in this foundation phase; richer graph expansion is deferred |

</threat_model>
