### Phase 11: Auto-Classification

**Goal:** Extend document classifier to extract Vendor, Product, Subsystem, and Version from filename patterns, directory hierarchy, and document metadata — no LLM dependency.

**Milestone:** v0.1.2
**Requirements:** CLASSIFY-01, CLASSIFY-02, CLASSIFY-03

**Success criteria — all met:**

1. ✅ A file named `OpenText WebReport Administrator Guide 23.4.pdf` is classified with `vendor=OpenText`, `product=WebReports`, `version=23.4`, `doc_type=admin_guide`
2. ✅ Classification fills gaps from PDF/DOCX metadata (title, subject, author, keywords) when filename is ambiguous
3. ✅ Existing `infer_product()`, `infer_doc_type()`, `classify()` signatures unchanged — backward compatible
4. ✅ All 585 tests pass; OTCS product detection still works as before

**Plans:** 2 plans — both executed 2026-05-25

**Delivered:**

- `infer_vendor()`: Detects OpenText from filename patterns, directory names, and product-to-vendor mapping (15 products mapped)
- `infer_subsystem()`: Detects subsystem from directory hierarchy (8 functional categories via filename patterns)
- `extract_document_metadata()`: Extract title/author/subject/keywords from PDF (PyMuPDF) and DOCX (python-docx)
- `enrich_classification()`: Gap-fills vendor/product/doc_type from document metadata (lowest precedence, never overrides explicit classification)
- Ingest pipeline stores vendor/subsystem in Qdrant chunk payload
- Bug fix: `DOC_TYPE_RULES` standard patterns now use word boundaries — no longer false-positive matches on "nist" substring in "Administrator"
- 72 classifier tests (58 existing + 7 metadata + 7 enrichment), all passing

## Milestone Summary

**Decimal Phases:**

- Phase 11.1: Vendor/Subsystem Integration Completion (inserted after Phase 11 for critical gap remediation)

**Key Decisions:**

- **Lazy cross-encoder loading:** Model not loaded at import time — first `rerank()` call triggers load. Decision saved ~500MB RAM and ~10s startup latency. (Phase 9)
- **Non-fatal health checks:** Server starts and accepts queries even when Qdrant or LM Studio are down — logs WARNING instead of crashing. Operators see health status via `kb-ingest check health`. (Phase 9)
- **Real qdrant_client imports in tests:** Removed 100+ lines of MagicMock sys.modules stubs. Tests import real model classes (Distance, PointStruct, FieldCondition) for proper enum comparison. (Phase 10)
- **Heuristics-first classification:** No LLM dependency for classification — deterministic rules for vendor/product/subsystem/doc_type/version. Metadata gap-filling at lowest precedence. (Phase 11)
- **Decimal phase insertion:** Phase 11.1 inserted for remediation of critical integration gap discovered during milestone audit. Avoids renumbering later phases. (Phase 11.1)

**Issues Resolved:**

- Cross-encoder lazy loading (saves 10s startup, 500MB RAM)
- MagicMock pollution in 3 test files
- Pre-flight health checks (no more silent startup failures)
- Vendor/subsystem integration gap (fields now visible in search results)

**Issues Deferred:**

- Integration tests for vendor/subsystem filtering (requires qdrant_client dependency in test environment)
- UAT showed MCP client/harness limitation in displaying full metadata fields (environmental, not code defect)

**Technical Debt Incurred:**

- test_reranker_lazy.py: Tests exist but fail in some environments due to conftest.py patching order (Phase 9)
- UAT revealed MCP client/harness does not display full JSON response structure — consider adding MCP inspector or dedicated test client (Phase 11.1)
